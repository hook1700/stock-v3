# 股票策略分析系统 v3

## 项目概述

这是一个基于股票数据的策略分析系统，旨在通过自动化策略执行帮助用户筛选符合特定投资逻辑的股票。系统支持多种投资策略（短线、中线、长线），并提供完整的股票数据获取、策略执行、结果记录功能。

## 技术栈

- **数据获取**: Python (BaoStock + AKShare)
- **后端服务**: Golang
- **前端界面**: Vue.js
- **数据库**: PostgreSQL
- **部署环境**: 腾讯云

## 核心功能需求

### 1. 股票列表展示
- 支持股票名称、股票ID筛选
- 支持多条件排序和组合排序
- 支持日期筛选
- 点击搜索后调用接口检索（非默认检索）

### 2. 策略自动执行
- 每日17:45自动拉取最新股票信息
- 执行预设策略，筛选符合逻辑的股票
- 提供买入点、卖出点和逻辑说明
- 支持策略单独重跑（避免单策略失败影响整体）

### 3. 记录管理
- 记录每日选出的股票
- 按策略分类和日期记录
- 支持历史记录查询

### 4. 板块资金流统计
- 统计板块的流入流出数据
- 前端显示当日、5日、10日、15日、20日数据

## 选股前提条件

### 通用要求（所有策略必须满足）
- **流动性要求**: 日均成交额 ≥ 2亿（短线≥5亿更好）
- **风险控制**: 非ST、非退市风险、不在严重利空期
- **交易指导**: 提供买入价和止盈价

## 投资策略体系

### 短线策略（持仓1-10天）

#### 策略①: 均线回踩低吸
- **适用**: 趋势热点股、板块上升期
- **选股条件**: 股价在20日线上，20日线向上；近5-10日温和放量上涨→缩量回踩5/10日线
- **买点**: 回踩5/10日线收小阳/十字星，尾盘14:50确认不破均线
- **卖点**: +5%~10%分批止盈，-7%或跌破回踩K线最低价止损

#### 策略②: 突破缩量回踩
- **适用**: 横盘震荡后选择方向的票
- **选股条件**: 近15-30日箱体震荡；曾放量突破箱体/前高；回踩原压力转支撑缩量
- **买点**: 回踩不破平台、收盘企稳→尾盘买
- **卖点**: 再次上攻放量不涨停→分批止盈

#### 策略③: 强势股10日线反抽
- **适用**: 强于大盘的板块龙头，短期回调
- **选股条件**: 近20日涨幅 > 大盘；回调至10/20日线出现承接
- **买点**: 第一次回踩10/20日线收阳→尾盘买
- **卖点**: 反弹至前高附近滞涨→卖出

### 中线策略（持仓1-3个月）

#### 策略①: 行业成长 + 均线多头
- **适用**: 行业景气（新能源/半导体/军工/出海制造/消费复苏）
- **选股条件**: 近两季营收/净利同比↑，ROE≥10%；股价站上60日线，5>10>20>60
- **买点**: 突破阶段平台后缩量回踩20/60日线企稳
- **卖点**: 主升加速放巨量→分批减；有效跌破20/60日线

#### 策略②: 困境反转 / 业绩拐点
- **适用**: 业绩由差转好、订单/政策催化
- **选股条件**: 前期调整>30%；近期季报营收/净利拐点向上；行业有利好
- **买点**: 第一根放量中大阳突破底部盘整→回踩不破阳线半分位/60日线
- **卖点**: 估值回归/利好兑现放巨量滞涨→止盈

#### 策略③: 高股息红利慢牛
- **适用**: 震荡市/偏弱市，求稳健
- **选股条件**: 银行/电力/高速/运营商/部分资源股；股息率≥4%~5%，ROE稳定
- **买点**: 缩量回踩60日线或年线附近→分批买
- **卖点**: 股息率降至不合理+破趋势线；分红大幅削减/基本面变化

### 长线策略（持仓6月-数年）

#### 策略①: 优质白马龙头
- **适用**: 连续3年ROE≥15%，净利↑，现金流好
- **选股条件**: 行业格局稳定或长期空间大；无治理/财务污点
- **买点**: PE处于历史30%~60%分位；分2-3笔建仓
- **卖点**: 基本面恶化；极端高估（PE>历史90%分位）

#### 策略②: 红利再投收息策略
- **适用**: 不想频繁操作，重视现金流
- **选股条件**: 连续5年分红，股息率≥4.5%；盈利稳定、负债低
- **买点**: 除权后贴权回踩支撑/大盘回调低估区
- **卖点**: 分红突然取消/大降；公司基本面恶化

#### 策略③: 真成长PEG低吸
- **适用**: 能接受波动、愿研究行业和增速
- **选股条件**: 净利增速20%~40%，ROE≥15%；PEG < 1
- **买点**: 合理估值区分笔建仓；大盘系统性下跌可补
- **卖点**: 增速明显放缓（PEG>1.5以上持续）；行业逻辑破坏

## 项目架构建议

### 数据层设计
- **股票基础数据表**: 存储股票基本信息、历史价格等
- **策略执行结果表**: 记录每日策略执行结果
- **板块资金流表**: 存储板块资金流向数据
- **用户操作记录表**: 记录用户筛选和查看历史

### 服务层设计
- **数据采集服务**: Python脚本定时获取股票数据
- **策略计算服务**: Golang服务执行策略逻辑
- **API接口服务**: 提供前端数据接口
- **定时任务服务**: 管理每日自动执行任务

### 前端设计
- **股票列表页面**: 支持筛选、排序、搜索
- **策略结果页面**: 展示每日策略执行结果
- **板块分析页面**: 显示资金流向统计
- **历史记录页面**: 查询历史策略结果

## 开发优先级

1. **第一阶段**: 基础数据获取和存储架构
2. **第二阶段**: 核心策略算法实现
3. **第三阶段**: 前端界面开发
4. **第四阶段**: 定时任务和自动化
5. **第五阶段**: 优化和部署

## 注意事项

- K线数据不需要存储和显示，有需要时跳转到股票网站查看
- 项目需要在腾讯云部署，提供完整的部署流程
- 当前环境已有Python和Vue环境
- 板块信息优先使用AKShare的申万分类（BaoStock无板块信息）
