## 服务器环境要求

- **操作系统**: Ubuntu 20.04 / 22.04 LTS（推荐）
- **内存**: 至少 2GB（推荐 4GB+）
- **磁盘**: 至少 20GB 可用空间
- **网络**: 能访问外网（用于拉取 Docker 镜像）

## 安装 Docker 和 Docker Compose

### 第一步：更新软件源
```bash
sudo apt update
sudo apt upgrade -y
```

### 第二步：安装 Docker
```bash
# 方式一：使用 apt 安装（推荐，版本可能较旧但稳定）
sudo apt install -y docker.io

# 启动 Docker 并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
```

### 第三步：安装 Docker Compose

#### 方式一：使用 apt 安装（推荐）
```bash
sudo apt install -y docker-compose-v2

# 验证安装
docker compose version
```

> **注意**：Docker Compose V2 的命令是 `docker compose`（空格分隔），而不是 `docker-compose`（横线分隔）。
> 本项目的 `deploy.sh` 已兼容两种写法。

#### 方式二：手动下载最新版
```bash
# 下载最新版 docker-compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 赋予执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

#### 方式三：如果 GitHub 访问慢，使用国内镜像
```bash
sudo curl -L "https://ghproxy.com/https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version
```

## 配置 Docker 镜像加速

国内服务器访问 Docker Hub（`registry-1.docker.io`）经常超时，需要配置镜像加速器。

### 第一步：创建 Docker 守护进程配置文件
```bash
sudo mkdir -p /etc/docker
```

### 第二步：写入镜像加速配置

#### 腾讯云服务器（推荐）
```bash
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com"
  ]
}
EOF
```

#### 其他云服务器或自建机房
```bash
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com",
    "https://mirror.baidubce.com"
  ]
}
EOF
```

### 第三步：重新加载配置并重启 Docker
```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 第四步：验证镜像加速是否生效
```bash
docker info | grep -A 5 "Registry Mirrors"
```

**预期输出**：
```
Registry Mirrors:
  https://mirror.ccs.tencentyun.com/
```

## 部署系统

### 第一步：上传项目代码到服务器

#### 方式一：使用 Git（推荐）
```bash
# 在服务器上克隆项目
cd /home/way
git clone <你的仓库地址> stock-v3
cd stock-v3
```

#### 方式二：使用 SCP 上传
```bash
# 在本地电脑上执行
scp -r stock-v3 root@你的服务器IP:/home/way/
```

### 第二步：修改部署脚本权限
```bash
cd /home/way/stock-v3/deploy
chmod +x deploy.sh
```

### 第三步：执行部署
```bash
./deploy.sh --deploy
```

**部署过程说明**：
1. 检查部署环境（Docker、Docker Compose 是否安装）
2. 创建配置文件（`backend/config.yaml`、`frontend/.env.production`）
3. 构建 Docker 镜像（后端、前端、数据采集）
4. 启动服务（PostgreSQL、Redis、后端、前端）
5. 健康检查（检查服务是否正常运行）

**部署成功后访问**：
- 前端地址：<http://服务器IP:3000>
- 后端 API 地址：<http://服务器IP:8080>

## 部署脚本使用说明

```bash
# 完整部署（首次部署使用）
./deploy.sh --deploy

# 更新系统（代码修改后重新构建并重启）
./deploy.sh --update

# 停止系统
./deploy.sh --stop

# 重启系统
./deploy.sh --restart

# 查看日志
./deploy.sh --logs

# 备份数据
./deploy.sh --backup

# 清理系统（会删除所有容器、镜像和数据，慎用！）
./deploy.sh --clean
```

## 常见问题排查

### 问题 1：Docker 镜像拉取超时

**错误信息**：
```
failed to resolve source metadata for docker.io/library/python:3.10-slim: dial tcp: i/o timeout
```

**解决方案**：
1. 确认已配置 Docker 镜像加速（见上文）
3. 如果镜像加速仍然超时，手动拉取基础镜像：
   ```bash
   sudo docker pull python:3.10-slim
   sudo docker pull golang:1.21-alpine
   sudo docker pull node:18-alpine
   sudo docker pull nginx:alpine
   sudo docker pull postgres:15
   sudo docker pull redis:7-alpine
   ```
4. 拉取成功后再执行部署

---

### 问题 2：DNS 解析失败

**错误信息**：
```
lookup docker.mirrors.ustc.edu.cn on 127.0.0.53:53: no such host
```

**解决方案**：
修改 DNS 配置：
```bash
# 查看当前 DNS
cat /etc/resolv.conf

# 修改为腾讯云 DNS 或公共 DNS
sudo tee /etc/resolv.conf <<-'EOF'
nameserver 183.60.83.19
nameserver 183.60.82.98
EOF

# 或者使用公共 DNS
sudo tee /etc/resolv.conf <<-'EOF'
nameserver 114.114.114.114
nameserver 8.8.8.8
EOF
```

---

### 问题 3：Docker Compose 命令找不到

**错误信息**：
```
[ERROR] 命令 docker-compose 未安装，请先安装
```

**解决方案**：
1. 确认安装的是 `docker-compose-v2`（命令是 `docker compose`）
2. 本项目的 `deploy.sh` 已兼容两种写法，会自动检测
3. 如果还是报错，手动安装旧版 `docker-compose`：
   ```bash
   sudo apt install -y docker-compose
   ```

---

### 问题 4：端口被占用

**错误信息**：
```
Bind for 0.0.0.0:8080 failed: port is already allocated
```

**解决方案**：
1. 查看占用端口的进程：
   ```bash
   sudo netstat -tulpn | grep 8080
   ```
2. 停止占用端口的进程，或者修改 `docker-compose.yml` 中的端口映射

---

### 问题 5：数据库初始化失败

**错误信息**：
```
psql: error: connection to server on socket "/var/run/postgresql/.s.PGSQL.5432" failed
```

**解决方案**：
1. 等待数据库完全启动（可能需要 30 秒以上）
2. 查看数据库日志：
   ```bash
   docker logs stock-strategy-db
   ```
3. 如果数据库启动失败，检查 `docker-compose.yml` 中的配置是否正确

---

### 问题 6：前端无法访问后端 API

**现象**：前端页面能打开，但数据加载失败

**解决方案**：
1. 检查后端服务是否正常运行：
   ```bash
   curl http://localhost:8080/health
   ```
2. 检查前端环境变量 `VITE_API_BASE_URL` 是否配置正确
3. 如果是生产环境，需要将 `localhost` 改为服务器 IP 或域名
